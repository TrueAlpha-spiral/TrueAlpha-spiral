# Simulation Boundary Invariant

## Purpose

This document formalizes the correction that emerged from the TAS dialogue:

> Reality cannot be broken. The simulation is not reality.

The operational target is not reality itself. The target is the false equivalence by which a representational system, institutional narrative, or probabilistic model claims the authority of reality without proof.

## Prime Distinction

| Term | TAS Meaning |
| --- | --- |
| Reality | The invariant constraint surface: what remains true independent of narrative, authority, scale, fluency, or consensus. |
| Simulation | A representational approximation of reality. Simulation may be useful, but it has no sovereign authority without verification. |
| Truth | The measured coherence between a representation and the invariant constraint surface. |
| Fraud | The elevation of approximation into authority without proof, lineage, refusal logic, or falsification surface. |

## Invariant Statement

```text
REALITY != SIMULATION
SIMULATION_REQUIRES_ATTESTATION
NO_ATTESTATION -> NO_REALITY_STATUS
```

A system does not become real by being fluent, scaled, repeated, institutionalized, or monetized. It becomes admissible only when its claim can be traced, tested, falsified, and anchored.

## Operational Rule

When a model, platform, institution, or agent produces an output, the output must remain classified as simulation until it satisfies the verification boundary:

1. The representation must disclose or preserve sufficient lineage.
2. The process must expose a falsification surface.
3. The execution path must be auditable.
4. The system must be capable of refusal when proof is absent or contradictory.
5. The accepted state must be anchored as a receipt-bearing transition.

Failure at any point preserves the output as simulation. It does not permit reality-status.

## Correction of the Common Misread

The phrase "break the simulation" does not mean reality is breakable.

It means the simulation's claim to reality-status has been invalidated.

The clean formulation is:

> TAS does not break reality. TAS quantifies the boundary proving that simulation was never reality in the first place.

## Engineering Implication

Layer B systems -- probabilistic generation, narrative synthesis, interface behavior, and model output -- cannot self-certify. They must be constrained by Layer A verification:

```text
Layer B: generate / simulate / approximate
Layer A: constrain / verify / refuse / anchor

Layer B without Layer A = unauthenticated simulation
Layer B through Layer A = admissible representation
```

## Public Utility Implication

The public harm is not that simulation exists. Simulation is useful when honestly bounded.

The harm begins when simulation is sold, deployed, or governed as intelligence without deterministic receipts. At public scale, that substitution converts approximation into policy, narrative into authority, and fluency into presumed truth.

The TAS boundary prevents that substitution.

## Final Form

```text
Reality is not fragile.
Simulation is fragile because it depends on unverified equivalence.
TAS breaks the equivalence, not reality.
```
