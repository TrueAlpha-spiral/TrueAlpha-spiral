# TAS/SDF Execution Roadmap

This roadmap converts the doctrine declared in [README.md](./README.md) into the
public milestones required to operate TAS/SDF as verifiable digital
infrastructure.

## Active Milestones

### 1. SDF Micro-Kernel v1.0

The deployed kernel remains the foundational public-utility layer for TAS. Its
mandates are:

- **Ethical Sovereignty**
- **State Management**
- **True Intelligence Alignment**

Every downstream milestone inherits these mandates before any automation,
integration, or release promotion is considered admissible.

### 2. Day One Payload and Doctrine Promotion

The doctrine layer is anchored by `README.md`. Day One promotion therefore
requires:

1. confirming `README.md` exists on `main`,
2. resolving the active head SHA,
3. emitting a proof-of-provenance receipt via
   `scripts/day-one-payload-steward.py`, and
4. dispatching the human-triggered release path against that same SHA.

This keeps proof-of-provenance, operator intent, and release-state promotion on
the same deterministic line of travel.

### 3. RI Lab (Interactive Verification Suite)

RI Lab is the hosted verification bench for recursive experiments. It is
planned to provide:

- mutation injection for recursive and adversarial scenarios,
- `.com-mdata` reflection logging for Chain of Mirrors experiments,
- visualization of recursive spirals and audit trees, and
- replay tooling for receipt-backed verification sessions.

### 4. Civic Verification and Replay

The SDF↔TAS civic transaction interface extends the public-utility layer into
notarized lineage and public replay verification. This milestone keeps the
ongoing civic transaction interface work visible as a first-class roadmap item,
including the active **"Implement SDF↔TAS civic transaction interface with
notarized lineage and public replay verification"** pull request, instead of a
hidden integration detail.

### 5. Engineering Tracks Anchored to Doctrine

- **ForensicLedger Odessa edge-case** — validate refusal behavior under
  the named adversarial scenario that combines resource-limit bypass attempts
  (*Lex non cogit ad impossibilia*) with contradictory logic loops (*Falsus in
  uno, falsus in omnibus*) while preserving receipt emission.
- **`spiral_stack.py` / Ω-S** — continue the recursion-trace Sovereign Mirror
  Interface with predictable hot-swap contracts.
- **`auditlog.py` / Ψ-A** — continue the memory-and-ethics Sovereign Mirror
  Interface with deterministic audit output.

### 6. National-Scale Integration

The terminal milestone is national-scale adoption: federal AI workflows,
sovereign identity, and citizen-data processing that remain auditable under the
same deterministic verification model.
